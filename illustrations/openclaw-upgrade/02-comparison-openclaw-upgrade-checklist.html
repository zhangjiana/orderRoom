OpenClaw 升级前，先按场景做判断

同样是“升级”，不同用户的优先级完全不一样。先看自己属于哪类，再决定是马上升、排期升，还是继续观察。

场景 1
Qwen 旧认证

还在用旧 portal 方式接 Qwen，需要迁移到 Model Studio API key。

场景 2
多 provider / gateway

依赖 OpenAI-compatible、embeddings、tool calling 兼容性，且会频繁切 provider。

场景 3
插件 / MCP / memory

已经在重度使用插件运行时、MCP bridge、memory 搜索或 channel 集成。

场景 4
稳定单机场景

本地环境很稳，不依赖上述高级能力，也没有明确兼容性痛点。

动作 1
马上升到 v2026.3.28

Qwen 迁移已经不是“优化项”，而是接入路径调整。

动作 2
维护窗口升级

有明显收益，尤其是 provider 兼容、失败回退和网关链路。

动作 3
近期值得升级

如果你靠这些能力吃饭，稳定性补丁比表面功能更重要。

动作 4
继续观察 stable tag

没有明确痛点时，优先等下一个正式 release。

底线判断：优先追 tagged stable，不要把 main 2026.3.30 当成生产环境的默认升级目标。
